# Contributing to Shortly

We welcome contributions to Shortly! Whether it's reporting a bug, suggesting a feature, or writing code, your input is valued.

## Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally.
3.  **Create a new branch** for your feature or fix:
    ```bash
    git checkout -b feature/my-new-feature
    ```

## Development Workflow

1.  **Install dependencies**:
    ```bash
    pnpm install
    ```
2.  **Make your changes**. Ensure you follow the existing coding style.
3.  **Run the service locally** to verify functionality:
    ```bash
    npm run dev
    ```

## Code Style

- We use **TypeScript** for type safety. Please ensure there are no strict type errors.
- Follow the existing project structure (Controllers, Routes, Services).
- Use `async/await` for asynchronous operations.
- Ensure variable and function names are descriptive.

## Pull Requests

1.  Push your branch to your fork.
2.  Open a Pull Request against the `main` branch.
3.  Provide a clear description of your changes and why they are needed.

Thank you for contributing!
